Problem

The problem we set out to solve is to create a program which takes a video as input and output a panoramic image, created from the frames in the input video. Creating a panoramic image from a video is an important problem to solve because it allows for interesting content to be created from a video after it has been recorded. This problem is also interesting with its potential applications. It can be used to create a simple panorama, or even a fully spherical panorama. It can also be used to visualize the total angular coverage of a panning video. Another important aspect of this problem is analyzing the efficiency and effectiveness of panorama generators. It is important to determine how to produce a panorama with minimal computational power, yet still yield an acceptable result.

Approach

We implemented the approach described by Zhu et al. [3] for panorama generation. We also tackled an open question mentioned in the paper. The question we would like to explore is how to handle camera movement and how much tolerance the system has to these movements. This is important because faster movements could cause blurring, reducing sharpness, and reduce the overlap between frames.

Challenges Encountered

  • How to efficiently and accurately select frames
  • How to deal with inconsistencies in videos (e.g. changing lighting)
  • Determining and implementing the fastest, yet most robust stitching algorithm

Key Implementation Points

  • Using optical flow to establish panning rate of video
  • Selecting frames based on that rate
  • Convert frames to a cylindrical projection
  • Detect and match features with customizable algorithm
  • Stitch frames together

Project written in Python, uses OpenCV and NumPy libraries
